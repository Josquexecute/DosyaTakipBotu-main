# Üretim Adayı (Production Candidate) Notları

Bu doküman, üretim adayı kabul sürecinin kalıcı kayıtlarını ve kontrol komutlarını özetler. Sürüme özgü değişiklikler için kök [CHANGELOG.md](../CHANGELOG.md) esastır.

## Üretim adayı kabul kapısı
Bir sürümün ofise dağıtılabilmesi için:
1. `npm run ci` tam zinciri (verify → smoke → denetimler → typecheck → build → davranış testleri → final office audit) geçer.
2. `npm run release:candidate-check` saha kabul kontrolü:
   - `package.json` ve `APP_VERSION` aynı sürümde.
   - Kurulum + taşınabilir EXE üretildi (`release/`).
   - `RELEASE_NOTES_v<sürüm>.md` ve SHA-256 hash dosyası mevcut.
   - Ofis hedef sürüm kaydı (`_HASARBOTU_OFFICE/office-version.json`) kontrol edilir.
3. `npm run live:preflight` salt-okunur canlı ön kontrol temiz.
4. `npm run live:backup-tracking` ile takip verisi yedeği alınabilir durumda.

## Sürüm bütünlüğü
- `package.json` = `APP_VERSION` = `package-lock.json` kök sürümü.
- `final-office-audit` beklenen sürümle uyumlu.
- Demo/ekran görüntüsü mock verileri güncel sürümü gösterir.
